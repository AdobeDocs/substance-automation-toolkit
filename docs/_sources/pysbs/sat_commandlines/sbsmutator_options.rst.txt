    <h2>
      <a name="edit">Subcommand edit</a>
    </h2>
    <h3>
      Description
    </h3>
    <p>
      Edit or instantiate the inputs, outputs or parameters of a graph in a Substance 3D Designer file.
    </p>
    <h3>
      Input file options
    </h3>
    <dl>
      <dt>
        <strong>--alias</strong>
        <em>&lt;aliasname:&frasl;&frasl;path&gt;</em>
      </dt>
      <dd>
        Define a custom alias path (e.g. 'myproject:&frasl;&frasl;path&frasl;to&frasl;myproject&frasl;library').
        This option is mandatory if your filter have a dependency with some custom alias.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--inputs, --input</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        Specify a Substance 3D Designer file to mutate with sbsmutator. Can be given multiple times
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--no-dependency</strong>
      </dt>
      <dd>
        Disable loading of dependencies.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--presets-path</strong>
        <em>&lt;path&gt;</em>
      </dt>
      <dd>
        Path to presets library (shorthand for '--alias sbs:&frasl;&frasl;&lt;path&gt;').
        This option is mandatory if your filter have a dependency to the presets library.
      </dd>
    </dl>
    <h3>
      Output file options
    </h3>
    <dl>
      <dt>
        <strong>--output-merge</strong>
      </dt>
      <dd>
        Merge generated content with output if a file is present, otherwise the file is overwritten.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-name</strong>
        <em>&lt;name&gt;</em>
        [default: &quot;{inputName}_edited&quot;]
      </dt>
      <dd>
        Set the output name of the generated files, if the correct file extension is not present it will be added.
        You can use the following patterns that will be replaced by the program when saving the result of the process:
        - {inputName} : replaced by the input filename.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-path</strong>
        <em>&lt;path&gt;</em>
      </dt>
      <dd>
        Set the output path for the generated files. By default the path is empty, i.e. the files will be saved in the current directory.
        You can use the following patterns that will be replaced by the program when saving the result of the process:
        - {inputPath} : replaced by the input filepath.
        
      </dd>
    </dl>
    <h3>
      Options for graph input&frasl;output
    </h3>
    <dl>
      <dt>
        <strong>--input-graph</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Select a specific graph from the loaded files.
        If you are editing multiple files, this options will apply to them all. If no graph are selected, the first graph found will be used.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-author</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        Set the author name of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-author-url</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        Set the author's URL of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-category</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Set the category of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-description</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        Set the description text of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-icon</strong>
        <em>&lt;path&gt;</em>
      </dt>
      <dd>
        Set the icon of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-label</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Set the label of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-name</strong>
        <em>&lt;name&gt;</em>
        [default: &quot;{inputGraph}&quot;]
      </dt>
      <dd>
        Name of the edited or generated graph to give in the output Substance 3D Designer file.
        Default='{inputGraph}'.
        You can use the following patterns that will be replaced by the program when saving the result of the process:
        -{inputGraph} replaced by the input graph name.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-physical-size</strong>
        <em>&lt;x&gt;,&lt;y&gt;,&lt;z&gt;</em>
      </dt>
      <dd>
        Set the physical size of the graph in the output Substance 3D Designer file, in cm.
        Vector components must be separated by comma.
        Example: --output-graph-physical-size 10.5,10.5,8
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-tags</strong>
        <em>&lt;tags;...&gt;</em>
      </dt>
      <dd>
        Set the tags of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output-graph-user-data</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        Set user data of the graph in the output substance 3Dfile.
      </dd>
    </dl>
    <h3>
      Specific options
    </h3>
    <dl>
      <dt>
        <strong>--connect-image</strong>
        <em>&lt;arg&gt;</em>
      </dt>
      <dd>
        Import image as resource in the new file, create a bitmap node and connect it to filter's input named.
        format of &lt;arg&gt;: &lt;input_name&gt;[@&lt;parameter_name&gt;@&lt;value&gt;]... 
        Available key-value parameters are:
        - &quot;path&quot; path of the image file
        - &quot;format&quot; cooked format of image (RAW&frasl;JPEG)
        - &quot;level&quot; cooking compression level of image (0 to 1)
        - &quot;tiling&quot; tiling of image activation (0 or 1)
        - &quot;absolute-size&quot; adapt image resolution relative to the parent or keep the absolute image size (0 or 1).
        - &quot;bit-depth&quot; set the bit depth of the created image node.Can be set to &quot;auto&quot;, &quot;8&quot;, &quot;16&quot;, &quot;16f&quot;, &quot;32f&quot;.
        You must provide the &quot;path&quot; parameter, so &lt;arg&gt; must look at least like &lt;input_name&gt;@path@&lt;actual_path_to_image_file&gt;
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--connect-input</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Create an input node and connect it to the input of the same identifier in the created instance filter. Only valid together with option --instantiate.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--hide-parameter</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Specifically hide parameter with identifier &lt;name&gt; of the filter. Only valid together with option --instantiate.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--hide-parameters</strong>
      </dt>
      <dd>
        Hide all specific parameters of the filter. Implies --remove-all-presets. Only valid together with option --instantiate.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--insert-sbsprs</strong>
        <em>&lt;path&gt;</em>
      </dt>
      <dd>
        Inject the presets contained in a *.sbsprs file into the output sbs.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--instantiate</strong>
      </dt>
      <dd>
        Switch to instantiation mode. In instantiation mode, the graph is not edited in-place; rather another graph is created with a instance node pointing to the input graph.
        Some operations can only work when using this option.
        Warning: The input of the instance are not exposed to the new graph by default.You need to use option --connect-input to expose them.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--remove-all-presets</strong>
      </dt>
      <dd>
        Remove all presets from the output graph.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--remove-output</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Remove the output node connected to filter's output those identifier is &lt;name&gt;.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--remove-preset</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Remove a preset from from an sbs file. &lt;name&gt; is the name of the preset to remove.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--set-bit-depth</strong>
        <em>&lt;name&gt;</em>
      </dt>
      <dd>
        Change the bit depth of the graph. &lt;name&gt; can be set to &quot;8&quot; (or &quot;int8&quot;), &quot;16&quot; (or &quot;int16&quot;), &quot;16f&quot; (or &quot;float16&quot;) or &quot;32f&quot; (or &quot;float32&quot;).
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--set-default-value</strong>
        <em>&lt;arg&gt;</em>
      </dt>
      <dd>
        Set default value of input. Format of &lt;arg&gt;: &lt;input_identifier&gt;@&lt;default_value&gt; Please note that the value is clamped between min and max.
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--switch-to-constant</strong>
        <em>&lt;arg&gt;</em>
      </dt>
      <dd>
        Remove input and switch all places using this input to a constant value. format of '&lt;arg&gt;': &lt;input_identifier&gt;
                        or &lt;input_identifier&gt;@&lt;value&gt;
        
      </dd>
    </dl>
