    <h2>
      <a name="AmbientOcclusion.Raytraced">Subcommand AmbientOcclusion.Raytraced</a>
    </h2>
    <h3>
      Description
    </h3>
    <p>
      Computes soft shadowing in occluded areas and stores the ambient lighting intensities in a grayscale image, where brighter areas receive more ambient light.
      
    </p>
    <h3>
      Common options
    </h3>
    <dl>
      <dt>
        <strong>--backends</strong>
        <em>&lt;string,...&gt;</em>
      </dt>
      <dd>
        Specifies the prioritization order of the raytracing backends, from first to last, as an ordered list. Defaults to the &quot;SAL&quot; GPU backend if unused.
        Available backends:
          - GPU: SAL
          - CPU: Embree, SoRa
        E.g.: --backends SAL,SoRa,Embree
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--base.uv_set</strong>
        <em>&lt;uint&gt;</em>
        [default: &quot;0&quot;]
      </dt>
      <dd>
        The target UV set of the meshes in the input scene.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--cpu</strong>
      </dt>
      <dd>
        Run the bakers exclusively on the CPU.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--enable_mip_diffusion</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Applies a diffusion post-process to the baked image, after padding.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--inputs</strong>
        <em>&lt;string;string;string;...&gt;</em>
      </dt>
      <dd>
        The 3D scene file(s) to process, which hold the low poly meshes.
        Alternatively, a list of paths may be provided without the '--inputs' option at the end of the command.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--name_suffix_high</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;_high&quot;]
      </dt>
      <dd>
        Suffix used by the high poly meshes in the scene.
        Used by the options matching meshes by name, such as &quot;projection.mesh_match_mode&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--name_suffix_low</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;_low&quot;]
      </dt>
      <dd>
        Suffix used by the low poly meshes in the scene.
        Used by the options matching meshes by name, such as &quot;projection.mesh_match_mode&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output_format</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;png&quot;]
      </dt>
      <dd>
        Format to use for the output image file.
        Supported formats: surface, dds, bmp, jpg, jif, jpeg, jpe, png, tga, targa, tif, tiff, psd, psb, hdr, exr, webp.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output_name</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;{scenename}_{bakername}&quot;]
      </dt>
      <dd>
        Output name of the output image files, without the extension. Macros may be used to tailor the name to the context of a bake.
        Available macros:
          - {scenename} : Replaced by the scene name before the last &quot;.&quot; (E.g.: scene.ext.fbx --&gt; scene.ext)
          - {bakername} : Replaced by the current baker's name
          - {udim} : Replaced by the UDIM ID of the current tile (MARI convention).
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output_path</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;.&frasl;&quot;]
      </dt>
      <dd>
        Path of the destination directory for the output image files.
        Available macros:
         - {inputPath} : Replaced by the path of the directory hosting the first input 3D scene file.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output_size</strong>
        <em>&lt;uint&gt;,&lt;uint&gt;</em>
        [default: &quot;2048,2048&quot;]
      </dt>
      <dd>
        Size of the output image files, in pixels.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--padding_radius</strong>
        <em>&lt;uint&gt;</em>
        [default: &quot;2&quot;]
      </dt>
      <dd>
        Width of the dilation post-process applied to the borders of UV islands in the output images, in pixels. Dilation is applied before diffusion.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--per_fragment_binormal</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Selects the source of data for computing the binormal of the tangent frame:
          - true : fragment shader.
          - false : vertex shader.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--recompute_tangents</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Forces the recomputation of tangents, dismissing those from the mesh. Computed tangents use the MikkT standard.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--selected_meshes</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        Selects meshes to include in the baking process.
        Expects the full path of a selected mesh in the input scene(s). If unused or empty, all meshes are selected. Use 'info' subcommand to list meshes and their full paths in an input scene.
        The option should be repeated for each selected mesh.
        E.g.: --selected_meshes &quot;&frasl;root&frasl;firstMesh&quot; --selected_meshes &quot;&frasl;root&frasl;secondMesh&quot;
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--udim</strong>
        <em>&lt;int&gt;</em>
        [default: &quot;1001&quot;]
      </dt>
      <dd>
        Coordinates of the UV tile to compute, as a UDIM ID using the MARI convention.
        E.g.: &quot;1022&quot;
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--use_cache</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;yes&quot;]
      </dt>
      <dd>
        Keep the meshes in memory between bakes.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--uv_tiles</strong>
        <em>&lt;int&gt;,&lt;int&gt;</em>
        [default: &quot;uint2(0,0)&quot;]
      </dt>
      <dd>
        Coordinates of the UV tile(s) to compute, as two zero-based indices.
        E.g.: &quot;1,2&quot;
        
      </dd>
    </dl>
    <h3>
      Projection options
    </h3>
    <dl>
      <dt>
        <strong>--cage_scene_path</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        The path of the 3D scene hosting the meshes that rays are casted from.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--high_scene_paths</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        The path(s) of the 3D scene(s) from which information should be baked onto the input scene.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.cull_backfaces</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Ignore backfacing triangles when trying to match low poly and high poly meshes.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.hit_strategy</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;inward&quot;]
      </dt>
      <dd>
        The method of selecting the intersection of rays with high poly meshes.
        Available options:
          - inward: Suited to capture floater geometry
          - closest_from_source: Take extra steps to select the closest low poly point
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.max_depth</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;0.01&quot;]
      </dt>
      <dd>
        Maximum rear distance for raytracing. I.e., the distance behind the surface of the low poly mesh.
        This option has no effect when the &quot;use_cage&quot; option is set to &quot;true&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.max_height</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;0.01&quot;]
      </dt>
      <dd>
        Maximum frontal distance for raytracing. I.e., the distance in front of the surface of the low poly mesh.
        This option has no effect when the &quot;use_cage&quot; option is set to &quot;true&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.mesh_match_mode</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;match_all&quot;]
      </dt>
      <dd>
        The method of matching the low poly and high poly meshes.
        - match_all : Intersections with any mesh in the high poly scene are used.
        - match_mesh_name : Only intersections with meshes of the same name than the low poly are used. E.g.: the &quot;clock_low&quot; mesh is only matched with &quot;clock_high&quot; mesh.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.normalized_distance</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Interpret the max distances as a factor of the mesh bounding box.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.offset_map_path</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        The path of a grayscale texture which values are added to the distance or the rays' source.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.sampling_rate</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;none&quot;]
      </dt>
      <dd>
        The supersampling ratio used to improve the quality of the output images. &quot;none&quot; means no supersampling is applied.
        Available options:
          - none
          - 2x2
          - 4x4
          - 8x8
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.skew_map_invert</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Invert the values of the skew map, such that bright areas in the skew map describe averaged directions while dark areas describe straightened directions.
        This option has no effect when the &quot;skew_correction&quot; option is not provided or set to &quot;false&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.skew_map_path</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        The path of the skew map.
        This option has no effect when the &quot;skew_correction&quot; option is not provided or set to &quot;false&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.smooth_normals</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Compute the rays' directions based on the averaged normals of the low poly mesh.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--skew_correction</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Enable the straightening of the rays' directions based on a grayscale texture: the skew map, which is used to specify the areas of the low poly meshes where the direction of rays should be smoothed (dark areas) or straightened (bright areas).
        This is useful to avoid projection deformation.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--use_cage</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Enable the use of a dedicated scene which meshes are used as a source for the projection rays.
        When set to &quot;false&quot;, the &quot;projection.max_depth&quot; and &quot;projection.max_height&quot; options are used to set arbitrary values for all meshes.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--use_lowdef_as_highdef</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Use the low poly scene as the high poly scene.
        This is meant for using most bakers without adding extra detail from other scenes.
        
      </dd>
    </dl>
    <h3>
      Baker options
    </h3>
    <dl>
      <dt>
        <strong>--attenuation</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;linear&quot;]
      </dt>
      <dd>
        The method of attenuation applied to occlusion shadows according to the occluder distance.
        Available options:
          - none
          - smooth
          - linear
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--culling_mode</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;never&quot;]
      </dt>
      <dd>
        The rule followed by occlusion rays for ignoring backfacing triangles in the high poly meshes.
        Available options:
          - never : No backfaces are ignored.
          - always : All backfaces are ignored.
          - match_mesh_name : Backfaces are only ignored for meshes which name includes the &quot;_ignorebf&quot; dedicated suffix.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--enable_ground_plane</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        If enabled, adds an infinite occluding plane at the lowest mesh point in the scene.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--ground_offset</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;0&quot;]
      </dt>
      <dd>
        Vertical offset applied to the ground plane, relative to the lowest mesh point in the scene.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--normal_map_orientation</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;directx&quot;]
      </dt>
      <dd>
        The orientation of the normal map.
        Only used in combination with the &quot;normal_map_path&quot; option.
        Available options:
          - opengl
          - directx
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--normal_map_path</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        The path of the external normal map providing additional surface details.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--normal_map_space</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;tangent_space&quot;]
      </dt>
      <dd>
        The coordinate space used by the normal map.
        Only used in combination with the &quot;normal_map_path&quot; option.
        Available options:
          - world_space
          - tangent_space
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.max_distance</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;1&quot;]
      </dt>
      <dd>
        Maximum distance of other detected surfaces. Surfaces farther than that distance are ignored.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.mesh_match_mode</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;match_all&quot;]
      </dt>
      <dd>
        The rule for detecting meshes in the high poly scene.
        Available options:
          - match_all : all meshes in the high poly scene are detected.
          - match_mesh_name : only meshes in the high poly scene with the same name as the source mesh in the low poly scene are detected.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.min_distance</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;1e-05&quot;]
      </dt>
      <dd>
        Minimum distance of other detected surfaces. Surfaces closer than that distance are ignored.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.normalized_distance</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Interpret the &quot;min_distance&quot; and &quot;max_distance&quot; values as multipliers of the mesh bounding box.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.sample_count</strong>
        <em>&lt;uint&gt;</em>
        [default: &quot;64&quot;]
      </dt>
      <dd>
        Number of secondary rays.
        Range: [1; 256]
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.sample_distribution</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;cosine&quot;]

    <h2>
      <a name="Height.Raytraced">Subcommand Height.Raytraced</a>
    </h2>
    <h3>
      Description
    </h3>
    <p>
      Extracts the vertical position of a mesh surface and stores it in a grayscale image, where higher areas are brighter and lower areas are darker.
      
    </p>
    <h3>
      Common options
    </h3>
    <dl>
      <dt>
        <strong>--backends</strong>
        <em>&lt;string,...&gt;</em>
      </dt>
      <dd>
        Specifies the prioritization order of the raytracing backends, from first to last, as an ordered list. Defaults to the &quot;SAL&quot; GPU backend if unused.
        Available backends:
          - GPU: SAL
          - CPU: Embree, SoRa
        E.g.: --backends SAL,SoRa,Embree
        
      </dd>
    </dl>
    <dl>

      </dd>
    </dl>
    <dl>

    <h2>
      <a name="Opacity.Raytraced">Subcommand Opacity.Raytraced</a>
    </h2>
    <h3>
      Description
    </h3>
    <p>
      Computes an opacity map where ray hits are white and ray misses are black. I.e., any ray hitting a mesh surface in the high poly scene will result in a white pixel.

    <h2>
      <a name="Thickness.Raytraced">Subcommand Thickness.Raytraced</a>
    </h2>
    <h3>
      Description
    </h3>
    <p>
      Computes the thickness of the meshes in the high poly scene(s) and stores it in a grayscale image, where brighter areas represent thicker volumes.
      The computation is very similar to the AmbientOcclusion.Raytraced baker, except the rays are cast in the direction opposite of the surface normal (I.e., inward) and hit an opposite surface.
      
    </p>

        <strong>--projection.cull_backfaces</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Ignore backfacing triangles when trying to match low poly and high poly meshes.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.hit_strategy</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;inward&quot;]
      </dt>
      <dd>
        The method of selecting the intersection of rays with high poly meshes.
        Available options:
          - inward: Suited to capture floater geometry
          - closest_from_source: Take extra steps to select the closest low poly point
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.max_depth</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;0.01&quot;]
      </dt>
      <dd>
        Maximum rear distance for raytracing. I.e., the distance behind the surface of the low poly mesh.
        This option has no effect when the &quot;use_cage&quot; option is set to &quot;true&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.max_height</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;0.01&quot;]
      </dt>
      <dd>
        Maximum frontal distance for raytracing. I.e., the distance in front of the surface of the low poly mesh.
        This option has no effect when the &quot;use_cage&quot; option is set to &quot;true&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.mesh_match_mode</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;match_all&quot;]
      </dt>
      <dd>
        The method of matching the low poly and high poly meshes.
        - match_all : Intersections with any mesh in the high poly scene are used.
        - match_mesh_name : Only intersections with meshes of the same name than the low poly are used. E.g.: the &quot;clock_low&quot; mesh is only matched with &quot;clock_high&quot; mesh.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.normalized_distance</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Interpret the max distances as a factor of the mesh bounding box.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.offset_map_path</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        The path of a grayscale texture which values are added to the distance or the rays' source.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.sampling_rate</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;none&quot;]
      </dt>
      <dd>
        The supersampling ratio used to improve the quality of the output images. &quot;none&quot; means no supersampling is applied.
        Available options:
          - none
          - 2x2
          - 4x4
          - 8x8
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.skew_map_invert</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Invert the values of the skew map, such that bright areas in the skew map describe averaged directions while dark areas describe straightened directions.
        This option has no effect when the &quot;skew_correction&quot; option is not provided or set to &quot;false&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.skew_map_path</strong>
        <em>&lt;string&gt;</em>
      </dt>
      <dd>
        The path of the skew map.
        This option has no effect when the &quot;skew_correction&quot; option is not provided or set to &quot;false&quot;.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--projection.smooth_normals</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Compute the rays' directions based on the averaged normals of the low poly mesh.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--skew_correction</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Enable the straightening of the rays' directions based on a grayscale texture: the skew map, which is used to specify the areas of the low poly meshes where the direction of rays should be smoothed (dark areas) or straightened (bright areas).
        This is useful to avoid projection deformation.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--use_cage</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Enable the use of a dedicated scene which meshes are used as a source for the projection rays.
        When set to &quot;false&quot;, the &quot;projection.max_depth&quot; and &quot;projection.max_height&quot; options are used to set arbitrary values for all meshes.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--use_lowdef_as_highdef</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;false&quot;]
      </dt>
      <dd>
        Use the low poly scene as the high poly scene.
        This is meant for using most bakers without adding extra detail from other scenes.
        
      </dd>
    </dl>
    <h3>
      Baker options
    </h3>
    <dl>
      <dt>
        <strong>--maximize_range</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;min_max&quot;]
      </dt>
      <dd>
        The normalization method applied to the computed image, if any. I.e., how to map the values to a specified range.
        Available options:
          - ray_distance : Remap the values to the range of distances of ray hits.
          - min_max : Remap the values from 0 to 1 where 0 is the thinnest high poly mesh volume and 1 is the thickest.
          - none : No normalization is applied.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.max_distance</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;0.1&quot;]
      </dt>
      <dd>
        Maximum distance of other detected surfaces. Surfaces farther than that distance are ignored.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.mesh_match_mode</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;match_all&quot;]
      </dt>
      <dd>
        The rule for detecting meshes in the high poly scene.
        Available options:
          - match_all : all meshes in the high poly scene are detected.
          - match_mesh_name : only meshes in the high poly scene with the same name as the source mesh in the low poly scene are detected.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.min_distance</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;1e-05&quot;]
      </dt>
      <dd>
        Minimum distance of other detected surfaces. Surfaces closer than that distance are ignored.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.normalized_distance</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;true&quot;]
      </dt>
      <dd>
        Interpret the &quot;min_distance&quot; and &quot;max_distance&quot; values as multipliers of the mesh bounding box.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.sample_count</strong>
        <em>&lt;uint&gt;</em>
        [default: &quot;64&quot;]
      </dt>
      <dd>
        Number of secondary rays.
        Range: [1; 256]
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.sample_distribution</strong>
        <em>&lt;string&gt;</em>
        [default: &quot;cosine&quot;]
      </dt>
      <dd>
        The angular distribution of rays.
        Available options:
          - uniform
          - cosine
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--secondary.spread_angle</strong>
        <em>&lt;double&gt;</em>
        [default: &quot;180&quot;]
      </dt>
      <dd>
        The maximum spread angle of rays.
        
      </dd>
    </dl>

    <h2>
      <a name="run">Subcommand run</a>
    </h2>
    <h3>
      Description
    </h3>
    <p>
      Executes a list of bakers described in a JSON file.
      JSON execution plans may be exported from Substance 3D Designer's baking interface.
      
    </p>
    <h3>
      Batch options
    </h3>
    <dl>
      <dt>
        <strong>--backends</strong>
        <em>&lt;string,...&gt;</em>
      </dt>
      <dd>
        Specifies the prioritization order of the raytracing backends, from first to last, as an ordered list. Defaults to the &quot;SAL&quot; GPU backend if unused.
        Available backends:
          - GPU: SAL
          - CPU: Embree, SoRa
        E.g.: --backends SAL,SoRa,Embree
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--cpu</strong>
      </dt>
      <dd>
        Run the bakers exclusively on the CPU.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--json</strong>
        <em>&lt;path&gt;</em>
      </dt>
      <dd>
        The JSON file describing an execution plan, which contains a list of bakers with global and discrete options.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--use_cache</strong>
        <em>&lt;bool&gt;</em>
        [default: &quot;yes&quot;]
      </dt>
      <dd>
        Keep the meshes in memory between bakes.
        
      </dd>
    </dl>
    <h2>
      <a name="update">Subcommand update</a>
    </h2>
    <h3>
      Description
    </h3>
    <p>
      Updates a JSON file describing an execution plan to the format used by this version of the Substance 3D bakers.
      
    </p>
    <h3>
      Options
    </h3>
    <dl>
      <dt>
        <strong>--input</strong>
        <em>&lt;path&gt;</em>
      </dt>
      <dd>
        The JSON file to update.
        
      </dd>
    </dl>
    <dl>
      <dt>
        <strong>--output</strong>
        <em>&lt;path&gt;</em>
      </dt>
      <dd>
        Specify the path of the output JSON file.
        If not specified the input file is updated in place.
        
      </dd>
    </dl>
